Compreendendo os Atributos de Classe e ID

Tanto class quanto id são atributos HTML usados para identificar e estilizar qualquer elemento, mas eles servem a propósitos diferentes. class define um grupo de elementos, enquanto id identifica exclusivamente um único elemento.

Como desenvolvedor, você decide quais valores ou identificadores devem ser atribuídos a classes e IDs. Esses identificadores devem ser descritivos e seguir uma convenção de nomenclatura consistente. Por exemplo, "box" é um bom nome de classe para um contêiner genérico.

O div HTML tem uma classe box.

<div class="box"> ... </div>
e o seletor CSS é (note a notação de ponto) 
.box {
  margin: 20px 0;
  border: 1px solid rgb(0 0 0 / 10%);
  padding: 10px;
  background-color: #ddd; 
}

Este div tem duas classes atribuídas: box e highlight. Portanto, o div terá os estilos de ambas as classes aplicados.

<div class="box highlight"> ... </div>
e o seletor CSS para highlight é 
.highlight {
  background-color: #efefaa;
  border-radius: 25px;
  padding: 25px;
}

Algumas declarações (background-color e padding, de "box") são sobrescritas pelas mesmas declarações em "highlight" porque "highlight" aparece após "box" na cascata CSS.

Este é um div com um ID exclusivo "note" e a mesma classe "box".

<div id="note" class="box"> ... </div>
e o seletor CSS é 
#note {
  background-color: #333;
  color: white;
  width: fit-content;
}

Testando o ID note.

  • NÃO use o mesmo ID em mais de um elemento
  • NÃO use IDs para estilização que se aplica a muitos elementos
  • NÃO comece nomes de classe ou ID com números
  • NÃO use espaços em nomes de classe ou ID (use hífens ou sublinhados)
